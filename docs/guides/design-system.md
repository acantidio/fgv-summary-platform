# Design System Reference

The visual design is enforced through the `SYSTEM_PROMPT` in `render.js`, which instructs Claude Opus to use these exact tokens when generating subject pages. The hub page (`docs/index.html`) is generated by `buildHub()` in `render.js` using the same CSS variables inline.

---

## Typography

| Role | Font family | CSS variable |
|---|---|---|
| Display (headings, card titles) | DM Serif Display | `var(--font-display)` |
| Body | DM Sans | `var(--font-body)` |
| Mono (labels, badges, code) | JetBrains Mono | `var(--font-mono)` |

Loaded via Google Fonts. Defined in `SYSTEM_PROMPT` inside `render.js` and applied by the hub's inline CSS in `buildHub()`.

### Accent colors (per-subject, defined in `ACCENT` object in `render.js`)

| Key | Light bg | Main color | Status badge, top border |
|---|---|---|---|
| `purple` | `#EEEDFE` | `#534AB7` | ✓ |
| `amber` | `#FAEEDA` | `#854F0B` | ✓ |
| `teal` | `#E6F4F1` | `#0F6E56` | ✓ |
| `blue` | `#E6F1FB` | `#185FA5` | ✓ |
| `rose` | `#FDE8EC` | `#C81E3A` | ✓ |

Adding a new color: add an entry to the `ACCENT` object in `render.js`, update the accent mapping in `SYSTEM_PROMPT` inside `render.js`, and add the value to `VALID_COLORS` in `test.js`.

## Responsive Breakpoints

Three breakpoints, no framework:

| Breakpoint | Width | Layout changes |
|---|---|---|
| Desktop | `> 640px` | Hub: 2-column grid, larger padding |
| Mobile | `≤ 640px` | Hub: single column, reduced padding, smaller h1 |

The `640px` breakpoint handles both mobile and tablet in a single rule — anything under that width goes single-column.

## Component Reference

### Hub card (`.card`)

```
┌─────────────────────────────────┐  ← 4px top border in accent.main
│ [BADGE]  Em andamento           │  ← badge: mono 10px, accent.light bg
│                                 │
│ Subject Title                   │  ← DM Serif Display 20px
│ Description text here...        │  ← DM Sans 13px, text-secondary
└─────────────────────────────────┘
```

Hover: lifts 2px, shadow deepens, border darkens.

### Subject header (`.subject-header`)

```
┌─────────────────────────────────┐  ← 4px top border in accent.main
│ [BADGE]  Concluído              │
│                                 │
│ Full Subject Title              │  ← DM Serif Display clamp(24px, 4vw, 36px)
└─────────────────────────────────┘
```

### Content typography hierarchy

```
## Section          → DM Serif Display 22px, border-bottom
### Sub-section     → DM Sans 15px bold
#### Label          → DM Sans 13px bold, uppercase, text-secondary
```

### Callout types (semantic signals to the AI renderer)

The four callout types in enriched files act as semantic signals to `render.js`. Claude Opus interprets each and renders it with a distinct visual treatment — not a fixed CSS class. The intent of each type:

| Type | Intent | Default visual treatment |
|---|---|---|
| `[!SUMMARY]` | ◎ Resumo — 2–3 sentence cheat-sheet | Full-width hero block at page top, larger font, accent border |
| `[!EXAM]` | ⚠ Cai na Prova — exam-critical content | Amber alert banner, prominent background |
| `[!KEY]` | ◆ Conceito-Chave — key concept/framework | Purple definition card, bold term + explanation |
| `[!RECALL]` | ? Recall Ativo — self-test question | Teal question card, grouped at end |

## Modifying the Design

Subject pages are AI-generated. To make a global design change:

1. Edit `SYSTEM_PROMPT` in `render.js` — update the color tokens, typography rules, or adaptive component guidance
2. Re-render all subjects: `npm run render -- --all`
3. Run `open docs/index.html` to verify visually

To change only the hub page:

1. Edit `buildHub()` in `render.js`
2. Run `npm run render -- --all` (or any single slug to trigger hub rebuild)

No CSS preprocessor, no external build tools.
